# 📊 Power BI Dashboard Setup Guide
## PMO Risk & Burn-Rate Tracker

This guide walks you through building the executive dashboard from the three CSV files generated by the Python pipeline.

---

## Step 1 — Import the Data

1. Open **Power BI Desktop**
2. Click **Home → Get Data → Text/CSV**
3. Import all three files in `/output/`:

| File | Purpose in Power BI |
|------|---------------------|
| `fact_daily_telemetry.csv` | Main fact table (1,399 rows) |
| `dim_alert_log.csv` | Alert events dimension |
| `kpi_executive_summary.csv` | KPI cards and summary visuals |

4. In **Power Query Editor**, verify:
   - `date` column = **Date** type
   - All `_usd` columns = **Decimal Number**
   - `overrun_pct` = **Decimal Number**
   - `project_id`, `client`, `region` = **Text**

5. Click **Close & Apply**

---

## Step 2 — Create Relationships

In **Model View**, create these relationships:

```
fact_daily_telemetry.project_id  ──▶  kpi_executive_summary.project_id
fact_daily_telemetry.project_id  ──▶  dim_alert_log.project_id
```

Both are **Many-to-One** relationships.

---

## Step 3 — Create DAX Measures

In the **fact_daily_telemetry** table, create these measures:

```dax
-- Total Actual Spend
Total Actual Spend = SUM(fact_daily_telemetry[actual_daily_spend_usd])

-- Total Planned Spend
Total Planned Spend = SUM(fact_daily_telemetry[planned_daily_spend_usd])

-- Overall Burn Rate %
Burn Rate % = DIVIDE([Total Actual Spend], [Total Planned Spend]) - 1

-- Active Critical Alerts
Active Critical Alerts = 
    CALCULATE(
        COUNTROWS(dim_alert_log),
        dim_alert_log[alert_type] = "CRITICAL"
    )

-- Active Warnings
Active Warnings = 
    CALCULATE(
        COUNTROWS(dim_alert_log),
        dim_alert_log[alert_type] = "WARNING"
    )

-- Average Health Score
Avg Health Score = AVERAGE(kpi_executive_summary[health_score])

-- Budget Variance $
Budget Variance = [Total Actual Spend] - [Total Planned Spend]
```

## Step 4 — Build the Visuals

### Visual 1: Gantt Chart (Deployment Timelines)

> Use the **Gantt Chart** custom visual from AppSource.
> Search: "Gantt Chart by MAQ Software" (free)

**Settings:**
- **Task**: `client`
- **Start Date**: `project_start_date`
- **End Date**: `project_end_date`
- **Legend (Color)**: `deployment_status`
- **Tooltip**: `overrun_pct_display`, `health_score`

**Color mapping for deployment_status:**
| Status | Color |
|--------|-------|
| Planning | `#4A90D9` |
| Hardware Procurement | `#7B68EE` |
| Staging & Config | `#20B2AA` |
| Site Installation | `#FFA500` |
| UAT | `#FFD700` |
| Go-Live | `#00FF7F` |
| Monitoring | `#00D4FF` |
| Closed | `#696969` |

---

### Visual 2: Dual-Axis Burn Rate Chart

**Visual type:** Line and Clustered Column Chart  
**Data source:** `fact_daily_telemetry`

**Settings:**
- **X-axis**: `date` (set to Day granularity)
- **Column Values**: `planned_daily_spend_usd` (bars, color `#3A3F5C`)
- **Line Values**: `actual_daily_spend_usd` (line, color `#00D4FF`)
- **Legend**: `project_id`
- **Filters**: Add a **Project slicer** so users can drill into one project

**Formatting:**
- Enable **data labels** on the line series only
- Add a **constant line** at the planned daily average to show the budget baseline
- Title: `"Daily Burn Rate: Planned vs. Actual"`

---

### Visual 3: KPI Cards (Top Row)

Create **4 card visuals** in a row at the top of the dashboard:

| Card | Measure | Alert Color Rule |
|------|---------|-----------------|
| 🔴 Critical Overruns | `Active Critical Alerts` | Red if > 0 |
| 🟡 Warnings | `Active Warnings` | Yellow if > 0 |
| 💰 Budget Variance | `Budget Variance` | Red if > 0 |
| 💚 Avg Health Score | `Avg Health Score` | Green if > 85 |

**For conditional coloring:**  
Format → Conditional formatting → Font color → Rules:
- If value > 0 → Font color: `#FF4444` (red)
- If value ≤ 0 → Font color: `#00FF7F` (green)

---

### Visual 4: Project Status Matrix

**Visual type:** Matrix  
**Data source:** `kpi_executive_summary`

**Settings:**
- **Rows**: `project_id`, `client`
- **Columns**: (none)
- **Values**: `overrun_pct_display`, `health_score`, `status_flag`, `pct_timeline_display`

**Conditional formatting on `overrun_pct_display`:**
- Value > 10 → Background: `#FF4444` (red)
- Value 5–10 → Background: `#FFA500` (orange)
- Value < 5  → Background: `#1E4620` (dark green)

## Step 5 — Apply the Dark-Mode Theme

1. Go to **View → Themes → Browse for themes**
2. Create a `.json` file with this content and import it:

```json
{
  "name": "Botsync Dark",
  "dataColors": [
    "#00D4FF", "#FF4444", "#00FF7F", "#FFA500",
    "#7B68EE", "#FFD700", "#20B2AA", "#FF69B4"
  ],
  "background": "#0D1117",
  "foreground": "#C9D1D9",
  "tableAccent": "#00D4FF",
  "visualStyles": {
    "*": {
      "*": {
        "background": [{"color": {"solid": {"color": "#161B22"}}}],
        "border": [{"color": {"solid": {"color": "#30363D"}}}],
        "fontColor": [{"color": {"solid": {"color": "#C9D1D9"}}}]
      }
    }
  }
}
```

3. Set the **canvas background** to `#0D1117`
4. Set all visual backgrounds to `#161B22` (slightly lighter than canvas)
5. Set all borders to `#30363D`

## Power Query Notes

If any columns need reshaping after import, use these Power Query M formulas:

```m
// Convert overrun_pct to percentage display
= Table.AddColumn(Source, "Overrun Display", each 
    Text.From(Number.Round([overrun_pct] * 100, 1)) & "%")

// Categorize health score
= Table.AddColumn(Source, "Health Category", each 
    if [health_score] >= 90 then "Excellent"
    else if [health_score] >= 75 then "Good"
    else if [health_score] >= 60 then "At Risk"
    else "Critical")
```
